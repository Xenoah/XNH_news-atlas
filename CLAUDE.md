## Architecture Decisions

### Namespace Pattern (`window.NewsAtlas`)
No build tools required. All scripts load via `<script>` tags in dependency order. `window.NewsAtlas = window.NewsAtlas || {}` at the top of every file makes load order robust.

### 3-Tier Data Priority
`data.js` probes `localhost:8787/status` with a 2s timeout on init. If reachable ‚Üí `live` mode. Otherwise ‚Üí `static` (pre-fetched GitHub Actions JSON). User clicking ‚Üª sets `_gdeltActive = true` ‚Üí `gdelt` mode (browser-side GDELT fetch). Static JSON silently auto-refreshes every 5 minutes.

### Per-Article Events (v2 pipeline design)
v1: 1 GDELT query = 1 aggregated event (max 40 events total).
v2: each article in a query result = its own map event. 52 topics √ó 50 articles = up to 2,600 raw per run; after URL dedup and 7-day accumulation the pool grows to 600‚Äì1,500 unique events.

### 7-Day Rolling Accumulation
`fetch-news.py` loads existing `world-latest.json` at run start, merges new articles (skips existing URLs), prunes events older than 7 days, then re-sorts and writes. This means a single throttled or failed run doesn't wipe existing data.

### Attention Score (per-article)
```
attentionScore = 0.35√ópositionRatio + 0.40√ócategoryWeight + 0.25√ócoverage
```
`positionRatio` = 1.0 for first article in query (most relevant), 0.0 for last.
`coverage` = unique source domains in topic / 20, capped at 1.0.
`bubble: true` when `attentionScore ‚â• 0.93`.

### Stable Coordinate Jitter
`stable_jitter(url, lat, lng)` seeds `random.Random` with the URL's MD5 hash. Same article always appears at the same map position across refreshes. Prevents piling of same-country articles on one point.

### GDELT DOC 2.0 API Usage (per official docs)
- Valid URL params: `query`, `mode`, `maxrecords`, `format`, `timespan`, `sort`
- `sourcelang` and `sourcecountry` are query **operators** embedded in the `query` string, not URL parameters
- `mode=ArtList`, `sort=HybridRel`, `maxrecords` max 250
- `mode=artgeo` does not exist in DOC 2.0 (separate GEO 2.0 API)

### Map-Ready Event Coordination
`data.init()` may complete before MapLibre fires `load`. Solution: `app.js` sets `_pendingRender = true` on map errors and listens for `newsatlas:mapready` dispatched by `map.js` after the `load` handler completes.

### Mobile Drawer Pattern
On ‚â§ 768px, side panels are hidden. Event detail opens `#mobile-drawer` via CSS `transform: translateY()`. Always in DOM, off-screen until `.drawer-open` toggled.

## Known Issues & Limitations

1. **CORS on file://** ‚Äî Must serve from HTTP server (`python -m http.server`, `npx serve`). `file://` loading fails.

2. **GDELT Rate Limiting** ‚Äî GitHub Actions IP ranges can be rate limited by GDELT during high-traffic news events. When all topics return empty, the script exits 0 and preserves existing data. The next hourly run retries.

3. **Location Accuracy** ‚Äî Articles without a topic-fixed lat/lng use `sourcecountry` (the publishing outlet's country, not the story's country). A Reuters article about an earthquake in Japan may be placed at `US` if sourcecountry=US. Topics with hard-coded coords (most conflict/politics topics) are not affected.

4. **MapLibre Glyphs** ‚Äî Cluster count labels use `https://demotiles.maplibre.org/font/` ‚Äî not guaranteed uptime for production. Self-host for reliability.

5. **No Pagination** ‚Äî GDELT DOC 2.0 has no pagination; max 250 articles per query. For broader coverage, multiple queries with different timespans would be needed.

6. **summary Field** ‚Äî GDELT artlist mode returns no article body text. Summary is generated as `"Breaking via {domain}. Covered by N sources in {category}."` ‚Äî not actual article content.
